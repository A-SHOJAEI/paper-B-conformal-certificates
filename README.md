# A Split-Conformal Recovery-Deadline Certificate for Runtime Assurance

The conformal recovery-deadline certificate as a standalone object: a distribution-free, finite-sample upper bound on fault-recovery time from split conformal prediction, used to engage a runtime shield only when recovery is provably overdue. Demonstrated on a deterministic pendulum domain and on the spacecraft attitude testbed.

## Reproduce

Run from the **repository root** (this folder). The package layout is preserved
so `python -m package.module` resolves directly:

```bash
pip install -r requirements.txt
python -m conformal_cert.domains.pendulum  # pure-NumPy: pendulum certificate
python -m paper.conformal.tables           # regenerate LaTeX tables from evidence
python -m paper.conformal.figures          # regenerate calibration/pareto figs
python -m program.conformal_rta            # spacecraft certificate (needs Basilisk)
```

## Status

The pendulum certificate is pure NumPy and runs from a clean checkout. The table/figure generators read committed evidence (`evidence/conformal_cert/pendulum_domain.json`, `evidence/program/conformal_rta.json`) and are pure NumPy / matplotlib. A fresh spacecraft run (`program.conformal_rta`) needs Basilisk (`pip install bsk`); the committed evidence lets the paper reproduce without it.

### Trained checkpoints (not shipped)

Learned weights (`checkpoints/*.pt`, `checkpoints/commander_*.gguf`) are large binaries and are **not** included. Each is regenerated by its training entry point in this repo (the RMA students via the `controller.rma.*` training mains; the commander via the `commander.nl_pddl.*` training mains). The committed `evidence/*.json` files let the manuscript tables/figures reproduce **without** the weights.

### Basilisk

Fresh high-fidelity runs use the Basilisk 6-DOF simulator. Install with `pip install bsk`. Basilisk-gated code paths skip gracefully when it is absent, and the pure-NumPy reproduce commands above do not need it.
